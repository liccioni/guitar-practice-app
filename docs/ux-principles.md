# Fretline UX Principles

## 1. Sessions Must Start Fast
- The user should be able to get into practice with minimal decision overhead.
- Review steps are acceptable only when they increase confidence, not friction.
- Default flows should bias toward starting practice, not configuring endlessly.

## 2. The Builder Must Feel Creative
- The builder is for shaping a routine, not filling out a form.
- Editing should feel direct and musical.
- Hide advanced options until needed.
- Avoid administrative language, dense settings blocks, and multi-step data entry when a direct edit works.

## 3. Active Practice Must Feel Focused
- Active practice is the product center.
- Remove distractions while the user is playing.
- Controls should support practice, not compete with it.
- The player should always understand:
  - what they are doing now
  - what is coming next
  - how to pause, skip, or finish

## 4. Feedback Must Be Immediate
- XP, streaks, badges, drill completion, and progress should respond quickly to effort.
- Feedback should reinforce momentum, not interrupt it.
- Use short, visible reward cues instead of long celebratory detours.

## 5. Avoid Clutter and Over-Configuration
- If a screen needs explanation, it is probably carrying too much.
- Remove low-value controls before adding new ones.
- Favor strong defaults over broad configurability.
- Every visible control must justify its presence in the current moment.

## 6. Fretline Must Feel Like a Guitar Tool
- Screens should feel like practice gear, not an admin product.
- Language should stay concrete and musician-oriented.
- Use terms tied to practice flow, rhythm, tempo, cues, streaks, and repetition.
- Avoid enterprise-style dashboards, settings matrices, and generic productivity framing.
